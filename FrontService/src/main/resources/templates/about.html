Проект создан для автоматизации работ с сайтами Совместных закупок.

СТЭК технологий:
Frontend = HTML, Js
Beckend = Java 17, Spring boot 3.2.2, Spring Cloud Gateway, Auth JWT

Клиент через главную страницу взаимодействует с сервисом.
Выбрав нужный сервис клиент отправляет запрос на сервер
К заголовку запроса прикрепляется JWT токен.
Все запросы от клиенсткой стороны проходят через Spring Cloud Gateway.
Он разделяет запросы на обязательные к проверке токена или нет.
К примеру для Авторизации и Регистрации токен не нужен и Gateway пропускает эти запросы далее.
Запросы к Resource microservice все запросы проходят проверку валидации токена.
Если валидация успешна, то запрос проходит дальше, если нет, то возвращается ошибка 401 UNAUTHORIZED
Если токен "протух" то вызывается его обновление. Этот процесс скрыт от пользователя
